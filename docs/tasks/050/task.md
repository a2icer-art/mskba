# task_full_description
Нужно централизовать контроль доступа так, чтобы для чувствительных разделов и действий обязательно проверялись: статус пользователя (только confirmed) и достаточный role_level. Сейчас эти проверки распределены по контроллерам/сервисам и применяются не везде. Требуется определить единый механизм (например, middleware или policy/gate-обёртка), внедрить его в нужные группы маршрутов (в первую очередь админ‑раздел), а также сохранить совместимость с текущими проверками прав. Изменения должны быть минимальными и локальными, без переработки архитектуры.

# План выполнения задачи
1. Аудит точек доступа и текущих проверок confirmed/role_level. (subtasks/01_audit.md)
2. Спроектировать и добавить централизованную проверку (middleware) для confirmed + min role_level. (subtasks/02_middleware.md)
3. Подключить проверку к админ‑роутам и минимально скорректировать дублирующиеся проверки. (subtasks/03_routes.md)
4. Обновить документацию по доступам (если требуется). (subtasks/04_docs.md)
