task_init_description:
Админ: раздел "Пользователи" (управление пользователями и сброс подтверждений контактов)


task_short_description:
Добавить в админку отдельный раздел "Пользователи" (вне модерации) со списком пользователей, фильтрами и попапом редактирования; предусмотреть возможность сбрасывать подтверждение контактов.


full_description:
Сделать в админ панели отдельный раздел "Пользователи" (не в блоке модерации). Внутри — список всех пользователей с фильтрами и таблицей. Колонки: ID, login (кликабельный), роли, статус пользователя, когда изменён статус, кем изменён, когда зарегистрирован, как зарегистрирован. По клику на login открывается попап с возможностью редактирования. В попапе нужен функционал сброса подтверждений контактов (email/телефон/telegram и др.) для админа.

План выполнения задачи:
1) Спроектировать состав таблицы, фильтры и формат данных; определить источники (модель User, роли, статусы, registered_via).
2) Реализовать backend: контроллер админского списка пользователей, фильтры, загрузка данных, endpoint для сброса подтверждения контакта.
3) Реализовать UI: новая группа "Пользователи" в sidebar, таблица и попап редактирования с действиями.
4) Проверить права доступа (admin/editor), протестировать сценарии.
5) Обновить документацию по админ‑разделам и процессам.
