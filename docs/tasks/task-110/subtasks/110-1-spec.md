# 110-1-spec

Цель: зафиксировать состав таблицы, фильтров и сценарии админского раздела "Пользователи".

## Раздел и навигация
- Новый раздел админки: группа "Пользователи" (отдельно от "Модерация").
- Пункт меню: "Пользователи".

## Таблица пользователей (колонки)
1) ID
2) Login (кликабельный, открывает попап редактирования)
3) Роли (список alias/label)
4) Статус пользователя
5) Изменен статус (дата/время)
6) Кем изменен (admin/user)
7) Зарегистрирован (дата/время)
8) Как зарегистрирован (registered_via)

## Фильтры (минимальный набор)
- Поиск по login / id
- Роль (мультиселект)
- Статус пользователя
- registered_via
- Диапазон дат регистрации

## Попап редактирования
- Заголовок: login + ID
- Блок "Контакты": список контактов пользователя
  - тип, значение, подтвержден/не подтвержден
  - кнопка "Сбросить подтверждение" (только для подтвержденных)
- Доп. поля для редактирования (позже): статус пользователя, роли (если потребуется)

## Сценарии сброса подтверждения
- Админ может сбросить подтверждение контакта (confirmed_at = null)
- История фиксируется аудитом
- Сброс доступен только при наличии роли admin/editor (уточнить правила доступа)
