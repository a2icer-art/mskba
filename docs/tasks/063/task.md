# task_short_description

# task_full_description
Нужно добавить UI и хранение дефолтных настроек времени событий: lead time и минимальная длительность. Раздел должен быть доступен в /admin/settings, ссылка в sidebar в группе "Система" рядом с логами. Значения из настроек должны переопределять значения config/events.php (fallback). Доступ — только админ/роль с доступом к админке.

# План выполнения задачи
1. Добавить сущность/хранилище настроек событий (модель, миграция, дефолты). (subtasks/01_model.md)
2. Добавить маршрут и страницу /admin/settings с формой настроек. (subtasks/02_ui.md)
3. Обновить логику событий на использование новых настроек с fallback на config. (subtasks/03_logic.md)
4. Обновить навигацию админки: пункт "Настройки" в группе "Система". (subtasks/04_nav.md)
5. Обновить документацию задачи и статус, добавить алгоритм проверки. (subtasks/05_docs.md)

Алгоритм проверки:
1. Открыть /admin/settings, изменить lead time и min duration, сохранить.
2. Открыть попап "Новое событие" без площадки — ошибки должны учитывать новые значения.
3. При отсутствии сохраненных настроек используются значения из config/events.php.
