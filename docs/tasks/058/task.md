# task_full_description
Нужно включить логирование для доменных сущностей событий и бронирований по аналогии с уже существующим audit-логированием (Auditable + config/entity-logging.php). Также требуется расширить раздел /admin/logs: добавить вкладки для сущностей, у которых логирование уже реализовано через Auditable, но не отображается в админке, а также новые вкладки для событий/бронирований. Логи должны работать для create/update/delete, включая действия модерации бронирований.

Реализовано: Auditable для Event и EventBooking, добавлены в entity-logging и вкладки админки для событий и бронирований.
