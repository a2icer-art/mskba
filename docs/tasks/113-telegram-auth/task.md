# full_description
Реализовать вход через Telegram-бота с deep-link из попапа авторизации. При успешной авторизации, если контакт Telegram у пользователя ещё не подтверждён — подтверждать. Если пользователя нет — создать аккаунт и профиль (минимальные данные), добавить новый контакт типа Telegram, подтвердить его и авторизовать пользователя.

Логин генерировать по правилу: если известен Telegram username (nick) — пробовать login как username; если занят — пробовать tg_<username>; если занят — tg_<telegram_id>. Если username отсутствует — использовать tg_<telegram_id>. Пароль генерировать 8-значный и отправлять пользователю в боте (вместе с логином). В поле регистрации (registered_via) указывать Telegram. Возможность стандартной авторизации по логину/паролю сохраняется.

UI: кнопка «Войти через Telegram» в попапе авторизации (рядом со стандартной формой). При клике открывается бот с одноразовым токеном (TTL), который используется для подтверждения входа через сервер.

Бизнес-ограничения: пользователь остаётся неподтверждённым, пока не пройдёт модерацию; для отправки на модерацию должны быть заполнены данные профиля и быть хотя бы один подтверждённый контакт (уже реализовано).
