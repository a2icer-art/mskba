# task_full_description
Нужно внедрить централизованный механизм отправки сообщений по контактам. На первом этапе реализовать только отправку на email через SMTP. Остальные типы контактов (телефон, мессенджеры и т.п.) должны иметь интерфейс/заглушки, чтобы их можно было подключить позже без изменения внешнего API.

Требования:
- Добавить в /admin/settings настройки поддерживаемых каналов отправки.
- Для email: поля SMTP (host, port, username, password, encryption/transport, from email, from name), возможность включить/выключить канал.
- Реализовать сервис отправки, который выбирает нужный delivery по типу контакта.
- Использовать новый сервис в процессе подтверждения контакта (email).
- Для остальных типов контактов — заглушки с безопасным поведением (например, логирование/return false) без падений.

Ограничения:
- Не ломать текущий флоу подтверждения контакта.
- Следовать существующим паттернам домена Users/Contacts и структуре проекта.

# task_plan
1. Проанализировать текущие `AdminSettings` и способ хранения настроек доставок.
2. Спроектировать доменную доставку: интерфейс/контракт, резолвер, заглушки для остальных каналов.
3. Реализовать email‑доставку через SMTP и связать с настройками.
4. Расширить `/admin/settings`: поля SMTP, переключатель канала, валидация/сохранение.
5. Подключить отправку в процесс подтверждения контакта (email) и обновить документацию.
