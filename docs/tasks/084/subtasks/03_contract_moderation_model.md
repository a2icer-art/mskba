# 03_contract_moderation_model

## Решение
- Запросы на владельца/супервайзера оформляются через `ModerationRequest` с типом `venue_contract`.
- `entity_id` хранит id площадки, `submitted_by` — пользователь-заявитель.
- В `meta` сохраняются `contract_type` и комментарий заявителя.

## Доменная логика
- Добавлен use case `SubmitVenueContractModerationRequest` с проверками:
  - подтвержденный пользователь;
  - подтвержденная площадка;
  - для owner требуется роль `venue-admin`;
  - для supervisor — отсутствие активного контракта на площадке;
  - отсутствие активного контракта нужного типа;
  - отсутствие pending-заявки по данному типу.
