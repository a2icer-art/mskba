# 09_docs_checks

## Сценарии проверки
1. Убедиться, что в контрактах отображаются типы `создатель/владелец/супервайзер/сотрудник`, а старые типы мигрированы в `supervisor`.
2. На странице контрактов площадки:
   - проверить отображение блоков запросов владельца/супервайзера;
   - отправить запрос на владельца (роль `venue-admin`), убедиться в статусе `pending`.
3. В `/admin/contracts-moderation`:
   - увидеть заявку с типом `owner`;
   - подтвердить с набором прав и комментарием;
   - проверить создание контракта и системное уведомление.
4. Отправить запрос на супервайзера:
   - подтвердить, что для пользователя без активного контракта кнопка доступна;
   - в модерации разрешены только права бронирования.
5. Проверить уведомления:
   - изменение статуса модерации;
   - назначение, изменение прав, аннулирование контракта.
6. Проверить видимость раздела «Супервайзер» в sidebar при наличии прав `venue.supervisor.view/manage`.
