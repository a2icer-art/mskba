# task_init_description
Спланировать под каждую роль участника отдельную сущность со своими полями: у игрока поля про игровые характеристики, опыт, скиллы, у тренера — опыт и регалии, у судьи — свои и т.д. (нужно придумать предварительные). Далее при регистрации пользователя использовать сервис регистрации вместо прямого обращения к модели; в сервисе проверять выбранную роль участника и создавать запись соответствующей модели.

# task_short_description
