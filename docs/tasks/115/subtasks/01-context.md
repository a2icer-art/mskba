# Подзадача 01: Контекст и точки интеграции

## Текущие сущности
- Event / EventBooking (бронирование события).
- PaymentMethod (методы оплаты, снимок в бронировании).
- VenueSettings (настройки площадки).
- Контракты площадки и права на работу с бронированиями.

## Точки интеграции
- Использовать снимок методов оплаты и получателя, зафиксированный в бронировании.
- Проверки прав: подтверждение оплаты — только по активному контракту с правом подтверждения бронирования + admin.
- Ограничения по времени — через настройки площадки.

См. `docs/project/event_organizer_payment_flow.md` (разделы «Источники данных оплаты», «Роли и права», «Ограничения по времени`).
