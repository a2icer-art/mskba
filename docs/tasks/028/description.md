# task_init_description
Добавить процесс подтверждения контакта через код: при нажатии "Подтвердить" отправлять код на контакт (email/telegram/etc). На фронте кнопка блокируется до ответа, затем появляется поле ввода кода, кнопка меняется на "Отправить код повторно". На бэке пока без реальной отправки — генерировать и сохранять код в БД. Нужна сущность подтверждений (код, ttl, user_id, contact_id и доп. поля).

# task_short_description
Добавить процесс подтверждения контактов с кодом, UX-логикой и сущностью подтверждений.

# task_full_description
Реализовать подтверждение контактов через одноразовый код. При нажатии "Подтвердить" генерировать код и сохранять в БД (без реальной отправки), блокировать кнопку до ответа, затем показывать поле ввода кода. После первого запроса кнопка становится "Запросить новый код". Если действующий код еще не истек, при нажатии показывается таймер ожидания. Требуется сущность подтверждений с полями: contact_id, user_id, code, expires_at (ttl), attempts и аудит. Добавить серверные эндпоинты, лимит попыток и логику TTL: при превышении попыток поле ввода остается видимым, но disabled, кнопка подтверждения скрывается, и показывается сообщение об исчерпании попыток; при перезагрузке страницы состояние сохраняется. Для каналов доставки предусмотреть интерфейс и заглушки реализаций (email/phone/telegram/vk/other).
