# Задача 100

- task_name: Сохранить UI-состояние на страницах ошибок
- task_status: выполнено
- task_init_description: Страница системных ошибок (500, 404, 403 и тд) сбрасывает некоторые установки.
- task_short_description: Исправить сброс пользовательских установок на страницах системных ошибок (500/404/403 и др.).

## full_description
Ошибка проявляется на 404/403/500: пропадает авторизованное состояние в UI (в шапке «Аккаунт», открывается модалка логина). Нужно сохранить shared props (auth/navigation/participantRoles/messageCounters) на страницах ошибок.

Решение: добавить Route::fallback для 404 (через web middleware), а для 403/500 сохранять shared props в обработчике исключений.
