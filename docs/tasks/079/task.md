# task_full_description
Перевести обновление статусов бронирований и связанные уведомления на фоновые задачи
через scheduler и job, чтобы логика не зависела от открытия страниц или polling.
Нужно убрать побочные вызовы (например, запуск expiry‑проверок из HTTP‑контроллеров),
вынести периодические проверки и смену статусов в отдельный доменный сценарий,
обеспечить корректный запуск по расписанию, идемпотентность и безопасную повторяемость.
