# step-02
Доменный сценарий и job для обновления статусов.

# Описание
- Вынести периодическую логику в доменный сценарий.
- Подготовить job с идемпотентным выполнением и безопасным повтором.

# Решение
- Используем существующий доменный сервис `BookingPaymentExpiryService` как ядро сценария.
- Добавлен job `App\Jobs\RunBookingPaymentExpiryJob`, который вызывает `runIfDue()`.
- Job реализует `ShouldQueue`, чтобы запускаться через очередь (подходит для прод‑окружения).
