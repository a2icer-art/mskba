# step-05
Документация и проверка.

# Описание
- Описать новый realtime‑флоу и сценарии проверки.
- Зафиксировать, что polling отключен или оставлен как fallback.

# Проверка
1. Убедиться, что в `.env` заданы `BROADCAST_CONNECTION=reverb` и `REVERB_*` / `VITE_REVERB_*`.
2. Запустить сервер WebSocket: `php artisan reverb:start`.
3. Отправить сообщение из другого пользователя — список диалогов, счетчики и активный чат обновляются без polling.
4. Прочитать диалог на втором аккаунте — у отправителя проставляется отметка прочитанности.

# Примечания
- Таймер polling отключен, HTTP‑запросы остаются для загрузки истории и действий пользователя.
