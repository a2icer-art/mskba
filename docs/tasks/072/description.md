# task_init_description
Добавить возможность задавать срок ожидания оплаты за бронирование (по умолчанию 60 минут, 0 = бессрочно). Показать настройку в настройках площадки с переключателем часы/минуты (как для длительности аренды), вывести информацию заказчику в бронировании. По истечении срока автоматически переводить бронирование в соответствующий статус с указанием причины.

# task_short_description
Срок ожидания оплаты бронирования: настройка, отображение, авто‑статус по истечении.

# task_open_questions
- В какой статус переводить бронирование по истечении срока ожидания оплаты (например, cancelled) и какая точная формулировка причины?

# task_process_notes
- Статус по истечении срока: `cancelled`, причина: «вышел срок на оплату».
- Автоматизация без cron: запуск проверки при заходе на релевантные страницы (например, `/venues/*/bookings`, `/events/*`) с троттлингом (не чаще 1 раза в 5 минут) и ограничением партии.
- Дополнительно при переводе в статус «оплачено» проверять срок и отменять просроченные.
