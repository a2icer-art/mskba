# 095-4-bookings

Цель: связать комиссию с расчетом цены бронирования и отображением в UI/админке.

- При создании платежа и переводе в ожидание оплаты сохраняются базовая стоимость и комиссия в `payments.meta` с учетом режима (проценты/фикс).
- В админском UI бронирований показывается разбивка базы/комиссии и итоговая сумма (с учетом режима).
- Итоговая сумма используется для валидации частичной предоплаты.
