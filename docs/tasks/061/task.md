# task_full_description
Нужно вынести временные параметры событий (lead time и минимальная длительность) в единые дефолтные настройки и заменить все хардкод-значения в проекте. Значения должны использоваться для проверки времени при создании события и в других местах, где сейчас применяются фиксированные числа.

# План выполнения задачи
1. Найти все места с хардкодом времени для событий и собрать список замен. (subtasks/01_audit.md)
2. Добавить единый источник дефолтных настроек для событий (config или доменный класс). (subtasks/02_defaults.md)
3. Обновить проверки времени событий на использование дефолтов. (subtasks/03_apply.md)
4. Обновить документацию задачи и статус, добавить чеклист UI-проверки. (subtasks/04_docs.md)

Алгоритм проверки в UI:
1. Открыть попап "Новое событие" и установить длительность меньше `events.min_duration_minutes` — получить ошибку.
2. Убедиться, что при выборе площадки lead time берётся из настроек площадки, иначе используется `events.lead_time_minutes`.
