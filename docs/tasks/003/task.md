# full_description
Нужно реализовать механизм авторизации на фронте и бэкенде. При нажатии кнопки «Войти» в header должно открываться popup-окно с формой авторизации. После успешного входа требуется обновить связанные компоненты: текст на кнопке «Аккаунт» в header должен меняться на login. При неуспешной авторизации в popup должно отображаться уведомление с ошибкой. Решение должно быть минимальным, с учетом текущей структуры проекта и Inertia/Vue.

# plan
1. Проанализировать текущий header и определить место для кнопки «Войти»/«Аккаунт», а также способ отображения модального окна — `subtasks/step-001.md`.
2. Реализовать popup-форму авторизации на фронте и связать её с backend login (Laravel auth) — `subtasks/step-002.md`.
3. Обновить отображение header после успешной авторизации и добавить показ ошибки в popup при неуспешном входе — `subtasks/step-003.md`.
