# task_full_description
Реализовать бизнес‑логику принятия бронирования в работу и выбора порядка оплаты на уровне заявки.
При выборе постоплаты бронь не подтверждается автоматически — подтверждение выполняется отдельным действием менеджера.
Предоплата (полная/частичная) обрабатывается одинаково: оплата происходит вне системы, менеджер вручную переводит бронь в статус «оплачено».
Система фиксирует выбранный порядок оплаты, срок ожидания оплаты и комментарий, но не ведёт учёт частичных сумм.
UI должен дать менеджеру выбор порядка оплаты и срока ожидания, а при частичной предоплате — поле суммы (по умолчанию 50%).
Изменение порядка оплаты допускается до момента оплаты; после оплаты — запрещено.
