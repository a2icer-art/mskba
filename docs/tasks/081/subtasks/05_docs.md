# step-05
Документация и проверка.

# Итог
- Процессы в `docs/project/processes.md` уже отражают выбранную модель оплаты (постоплата без авто-подтверждения,
  предоплата/частичная как единый сценарий оплаты вне системы).
- Добавлены сценарии UI-проверки.

# UI-сценарии проверки
1. Открыть `/venues/<type>/<alias>/bookings` как менеджер площадки.
2. Для заявки со статусом `pending` нажать «Принять в работу».
   - Выбрать «Предоплата»: задать срок оплаты, сохранить — статус станет `awaiting_payment`,
     появится срок оплаты и уведомление.
   - Выбрать «Частичная предоплата»: проверить дефолт 50% от суммы, сохранить — статус `awaiting_payment`.
   - Выбрать «Постоплата»: срок оплаты и сумма скрыты, сохранить — статус остается `pending`.
3. Для постоплаты выполнить отдельное подтверждение («Подтвердить»), убедиться в статусе `approved`.
4. Для предоплаты отметить оплату («Оплачено»), проверить переход в `paid/approved` по режиму площадки.
5. Для постоплаты после подтверждения отметить «Оплачено» — статус остается `approved`,
   уведомление у участников обновляется.
