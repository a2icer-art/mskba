# full_description
Нужно убрать хранение методов оплаты на уровне контрактов. Методы оплаты остаются у площадок как сейчас, а для контрактов должен выбираться один из методов пользователя (носителя контракта). Для будущих сценариев (например, оплата участия в событии) должна применяться аналогичная логика: выбирается один из методов пользователя. Также нужно добавить свойство "метод по умолчанию" в PaymentMethod, которое автоматически устанавливается при добавлении первого метода пользователя. UI/валидация должны работать с новым выбором метода для контракта.

# План выполнения задачи
1. Проанализировать текущую схему PaymentMethod/Contract и места использования контрактных методов (бэкенд/фронт).
2. Добавить признак is_default в PaymentMethod, миграцию и правила авто-установки при первом методе пользователя.
3. Удалить зависимость контрактов от payment_methods (морф-связь/выбор), заменить на contract.payment_method_id (связь на метод пользователя).
4. Обновить UI управления контрактами: вместо CRUD методов — выбор метода пользователя (dropdown/список).
5. Обновить сервисы/валидации/форматирование и места использования контрактных методов.
6. Обновить документацию (docs/project/current_state.md) и проверить сценарии.
