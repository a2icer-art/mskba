# 105-2-verify

Цель: проверить ключевые сценарии участия.

## Чек-лист
- [x] Приглашение участника через админ/создатель: фильтр по роли, запись в invited.
- [x] Самозапись участника: confirmed при наличии лимита, reserve при превышении.
- [x] Принятие приглашения: confirmed или reserve при превышении лимита.
- [x] Изменение статуса организатором: confirmed/reserve/declined с причиной.
- [x] Запрет «повышения» статуса пользователем, если статус менял организатор.
- [x] Тексты статусов на публичной странице отображаются корректно.
- [x] Отображение счетчиков участников/резерва и стоимости участия.

## Фактическая проверка
- Статус: проверено в UI (подтверждено пользователем).
