# task_init_description
Переработать логику участия в событиях: заявки через pending, подтверждения/отклонения по источнику (пользователь/организатор), разрешить резерв и повторные заявки.

# task_short_description
Обновить участие в событиях: статус pending, источник заявки, подтверждения организатором или участником, резерв и повторное подтверждение с проверкой лимитов.
