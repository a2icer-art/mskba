# full_description
Нужно добавить пользователю в разделе /account (группа "Настройки") новый пункт "Платежная информация". Раздел должен использовать текущую инфраструктуру PaymentMethod (polymorphic owner) и работать для owner = User. Доступ к управлению методами оплаты должен быть у владельца аккаунта и у admin. Набор типов методов оплаты остаётся общим/централизованным (используются те же типы, что и у площадок/контрактов). Требуется обеспечить единый UX управления методами оплаты на стороне пользователя.

# План выполнения задачи
1. Проанализировать текущую реализацию PaymentMethod для площадок/контрактов (валидация, форматирование, UI) и определить точки переиспользования для пользователя.
2. Добавить связь paymentMethods() в app/Models/User.php, а также роуты/контроллер для раздела /account/settings/payment-info с доступом owner/admin.
3. Вынести общую валидацию/форматирование методов оплаты в переиспользуемый слой (helper/trait/service), чтобы не дублировать логику из VenuesController.
4. Реализовать страницу/секцию в resources/js/Pages/Account для управления методами оплаты пользователя (UI по аналогии с VenueSettings/VenueContracts).
5. Добавить пункт “Платежная информация” в sidebar группы “Настройки” и проверить UX/ошибки.
6. Обновить docs/project/current_state.md и зафиксировать изменения.
