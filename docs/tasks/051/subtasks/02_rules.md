# Подзадача
Правила обработки расписания, исключений и влияние на бронирования.

# Правила
- Одно расписание на площадку, таймзона фиксированная UTC+3.
- Недельные интервалы хранятся по дням недели; допускается несколько интервалов в день (перерывы = промежутки между интервалами).
- Интервал валиден при starts_at < ends_at.
- Интервалы внутри одного дня не пересекаются и не соприкасаются (по договоренности): [start, end) — конец не включается.
- Исключение на дату имеет приоритет над недельным расписанием:
  - Если is_closed = true, площадка недоступна в этот день.
  - Если is_closed = false, использовать интервалы из VenueScheduleExceptionInterval.
- Бронирование валидно, если:
  - полностью попадает в доступный интервал дня (weekly или exception), и
  - не пересекается с уже существующими бронированиями.
