# task_init_description
расписание площадок VenueSchedule - необходимо разработать новую сущность связанную с Venue из dev

# task_short_description
Спроектировать сущность VenueSchedule и связь с Venue для расписаний площадок.

# task_full_description
Нужно спроектировать сущность VenueSchedule, связанную с Venue (одно расписание на площадку). Расписание влияет на будущие бронирования, вместе с уже занятыми слотами. Таймзона фиксированная: UTC+3 (Москва). Необходимо предусмотреть перерывы внутри дня (несколько интервалов доступности).

# решение (варианты)
1) Weekly + exceptions: базовое недельное расписание (дни недели + интервалы) + таблица исключений для конкретных дат.
2) Rules + overrides (рекомендуется): правила по дням недели + overrides на конкретные даты (переопределение интервалов).
3) Слоты доступности по датам: хранить конкретные интервалы на даты (гибко, но объем данных больше).
