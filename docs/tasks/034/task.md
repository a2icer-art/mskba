# task_full_description
Нужно реализовать модерацию площадок по аналогии с модерацией пользователей. Требуется создать страницы Admin и backend-обработчики, чтобы администратор мог видеть заявки по площадкам, фильтровать по статусам, сортировать и принимать решения (подтвердить/отклонить). Логика должна использовать существующую инфраструктуру модерации: `moderation_requests`, статусы `pending/approved/rejected`, правила требований и единые сервисы. В интерфейсе должна быть таблица заявок с данными площадки, датами отправки/решения, ответственным, а также действиями. При отклонении — опциональная причина. Доступы: раздел Admin доступен при level > 10, модерация площадок — при level > 20.

# План выполнения задачи
1. Описать и реализовать правила модерации площадок (requirements) и подготовить выдачу данных для Admin. (docs/tasks/034/subtasks/step-001.md)
2. Реализовать страницы Admin для модерации площадок: список, фильтры, сортировка, действия approve/reject. (docs/tasks/034/subtasks/step-002.md)
3. Интегрировать ссылки в Admin‑меню, обновить документацию задачи и итоговые статусы. (docs/tasks/034/subtasks/step-003.md)
