# Task 111 — Счётчики модерации (moderation pending badges)

Исходный текст задачи:
Добавить счётчики «модерация — в ожидании» в хедер (Панель управления) и в элементы сайдбара группы модерации, показывающие общее количество заявок в статусе `pending` и, при необходимости, количество по типам сущностей (`user`, `venue`, `venue_contract`). Счётчики должны выглядеть и вести себя аналогично счётчику непрочитанных сообщений: показывать число, при >9 — `…`, использовать тот же стилистический компонент и цвет (rose/красный).

Краткое описание (task_short_description):
Добавить на сервере подсчёт `pending` заявок модерации и пробросить значения в Inertia navigation props; отобразить их в `MainHeader` и в сайдбаре (через `AdminNavigationService`), используя существующую логику `item.badge` и `formatSidebarBadge`.

Дополнительные требования:
- Подсчёт должен учитывать только статусы `pending` модели `App\Domain\Moderation\Models\ModerationRequest`.
- Предоставлять общее значение и, опционально, по типам сущностей для отдельных пунктов сайдбара.
- Проверять права: счётчики отображать только тем пользователям, у которых есть доступ к модерации.
- Минимальные изменения в проекте: расширить `AdminNavigationService::getMenuGroups()` или точку, где формируется `navigation` props, и обновить фронтендное отображение, используя уже существующие компоненты.
- Обновить документацию задачи и добавить `task.md` в папке задачи при старте выполнения.

Acceptance criteria:
- В `MainHeader` рядом с «Панель управления» появляется красный бейдж с числом pending-запросов (или `…` если >9).
- В сайдбаре, в группах модерации, у соответствующих пунктов отображается `item.badge` с количеством pending для типа сущности.
- Изменения покрыты кратким комментариям в коммите и добавлением `docs/tasks/task-111/task.md` при старте работы.
