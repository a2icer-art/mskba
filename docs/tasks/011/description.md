# task_full_description
Проблема: task-ветки нужны для возврата к контексту и анализа истории, но их большое количество засоряет список веток и усложняет навигацию. Нужно определить единый подход: где хранить ветки (локально/на GitHub), какие ветки сохранять, и как это влияет на удобство команды и поддержку репозитория.

## Варианты
1. Хранить task-ветки только локально.
   - Плюсы: чистый список веток на GitHub, меньше «шума».
   - Минусы: на другом компьютере веток нет; при потере локальной истории восстановить сложно.

2. Хранить task-ветки локально и на GitHub.
   - Плюсы: доступ из любой среды, легко открыть старую ветку.
   - Минусы: много веток на удалённом, сложнее навигация.

3. Хранить task-ветки только на GitHub.
   - Плюсы: централизованная история, доступна всей команде.
   - Минусы: захламление удалённого, локально всё равно нужно обновлять список.

## Критерии решения
- Удобство возврата к старым задачам.
- Чистота списка веток в репозитории.
- Возможность работы с разных устройств.
- Минимизация ручного обслуживания веток.
