# task_full_description
Нужно учесть настройки площадки (lead time и min interval) в UI бронирований: предварительно блокировать неподходящие значения, показывать подсказки и корректные сообщения на клиенте. Источник настроек — venue_settings. Должен быть fallback на дефолты, если настройки отсутствуют.

# План выполнения задачи
1. Определить точки UI, где используется lead time и min interval (страница события и попап бронирования). (subtasks/01_audit.md)
2. Прокинуть настройки площадки в UI при бронировании. (subtasks/02_props.md)
3. Обновить фронт‑валидацию и подсказки с учетом настроек. (subtasks/03_ui_rules.md)
4. Обновить документацию задачи и статус. (subtasks/04_docs.md)

Алгоритм проверки в UI:
1. В попапе бронирования выбрать площадку с lead time/min interval ≠ дефолту.
2. Указать старт раньше допустимого lead time → появится клиентская ошибка.
3. Указать длительность меньше min interval → появится клиентская ошибка.
4. Исправить значения — кнопка становится активной, отправка проходит.
