task_short_description:
Добавить поддержку подтверждения контакта типа telegram в общем процессе верификации контактов.


full_description:
Нужно реализовать подтверждение контакта типа Telegram по механике одноразового токена, максимально похоже на текущую email-верификацию, но с учетом ограничений Telegram: бот не может написать пользователю первым. Процесс: пользователь в UI нажимает “Подтвердить”, получает deep-link на бота с токеном, в чате бота нажимает “Подтвердить”, бот вызывает callback/вебхук на сайт, токен валидируется и контакт подтверждается.

Токен: одноразовый, с высокой энтропией (128+ бит), хранится в БД в виде хэша, привязан к user_id и contact_id (или user_id), с TTL и проверкой попыток. Поведение при истечении срока — выдача нового токена. Подтверждение должно обновлять UI (автообновление статуса). Нужен вебхук для Telegram-бота и обработчик callback. Требуется минимальная безопасность (подпись, сравнение hash/secret по возможности), без нарушения требований Telegram.

План выполнения задачи:
1) Аудит текущей email-верификации и точек интеграции (ContactVerification, сервисы, UI, маршруты). (subtasks/109-1-audit.md)
2) Реализация Telegram-верификации: токен (hash/TTL/одноразовый), deep-link, выдача/обновление в ContactVerificationService. (subtasks/109-2-token-flow.md)
3) Telegram webhook и обработка callback подтверждения (валидация токена, подтверждение контакта). (subtasks/109-3-webhook.md)
4) UI аккаунта для Telegram: запуск подтверждения, статус, автообновление. (subtasks/109-4-ui.md)
5) Документация, ручная проверка и фиксация результата. (subtasks/109-5-docs.md)

Результат:
- Добавлен одноразовый Telegram токен (hash) в ContactVerification и миграция.
- Реализован deep-link на бота и подтверждение через webhook/callback.
- Добавлена сверка username/chat_id с value Telegram-контакта и нормализация формата (@username).
- UI аккаунта показывает подсказки и auto-refresh статуса.
- Обновлена проектная документация и deploy.md (webhook для prod).
