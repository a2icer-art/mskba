# task_full_description
Нужно выделить универсальную доменную сущность Payment, пригодную для разных сценариев оплаты. Сущность должна хранить сумму, валюту (с возможностью расширения), статус (created/pending/paid/cancelled/failed), ссылку на пользователя (плательщика) и ссылку на целевую сущность (polymorphic target). На первом этапе использовать связку с бронированиями как единственным кейсом, не ломая текущий функционал.
