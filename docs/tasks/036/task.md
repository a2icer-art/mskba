# task_init_description
редактирование пользователя (своего пользователя). Редактирование пользователя затрагивает модели User, UserProfile. Если пользователь не подтвержден - редактировать можно все поля UserProfile, если подтвержден только те, что не указаны как обязательные при отправке на модерацию. У модели User - редактировать можно следующие поля: password (валидация). Чтобы перейти в редактирование должна быть кнопка "редактировать" справа от заголовка в карточке h1 (там же где в площадках - добавить площадку)

# task_short_description
Добавить редактирование профиля и пароля: кнопка «редактировать» в карточке профиля, правила доступа по статусу подтверждения.

# task_full_description
Нужно добавить редактирование собственных данных пользователя. Редактирование затрагивает модели `User` и `UserProfile`. Если пользователь не подтвержден — доступны все поля профиля. Если подтвержден — доступны только поля профиля, которые не являются обязательными для модерации (см. требования в `UserModerationRules`). Для модели `User` разрешено редактировать пароль с валидацией. Вход в режим редактирования — кнопка «Редактировать» справа от заголовка в карточке (где сейчас `h1` на странице профиля). Процесс редактирования должен быть выполнен через сервис, который логирует изменения.

# План выполнения задачи
1. Подготовить сервис обновления профиля и пароля с логированием и правилами доступа. (docs/tasks/036/subtasks/step-001.md)
2. Добавить маршруты и контроллеры для обновления профиля и пароля через сервис. (docs/tasks/036/subtasks/step-002.md)
3. Обновить UI страницы аккаунта: кнопка «Редактировать», режим формы и ограничения по статусу. (docs/tasks/036/subtasks/step-003.md)
