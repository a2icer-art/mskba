# task_init_description
давай заведем новую задачу #set_task в которой мы продумаем как будет определяться метод оплаты бронирования площадки - у нас есть несколько сценариев - если у площадки есть супервайзер (агент) то нужно указывать/использовать его данные - эти данные будут показаны участникам которые создают событие и бронирование площадки. если супервайзера нет - оплата производится на прямую площадке. у площадки может быть владелец (подтвержденный) тогда можно использовать его данные. а если владельца нет что делать? если есть супервайзер, но нет владельца - как супервайзер поймет куда оплачивать? неужели нужно заводить доп. информацию по оплате для самой площадки? (дефолтную для fallback?)
2. нужно сделать интерфейс - в котором будет задаваться методы оплаты, например "СБП" - в этом методе должны быть следующие поля: номер телефона для перевода средств, отображаемое имя (для проверки в процессе перевода по СБП, обычно Имя и Инициалы). Также нужно поддерживать возможность нескольких вариантов оплаты - оплата с баланса пользователя внутри системы (mskba), оплата через эквайринг (пока заглушка). у методов оплаты должен быть статус - активный/неактивный

# task_short_description
Определить правила выбора реквизитов оплаты бронирования (супервайзер/владелец/площадка/фолбэк) и спроектировать интерфейс управления методами оплаты (СБП, баланс, эквайринг) со статусами.
