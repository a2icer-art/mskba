# 02_service.md
- Сервис `BookingNotificationService` формирует системные сообщения по статусам бронирований.
- Получатели: создатель бронирования + пользователи площадки с правами `venue.booking.confirm`/`venue.booking.cancel`.
- Создается (или используется) системный диалог по бронированию: `type=system`, `context_type=EventBooking::class`, `context_id=booking.id`, `context_label=Бронирование №CODE`.
- В сообщение пишется:
  - `title` — системный текст статуса.
  - `body` — комментарий модерации (если есть).
  - `link_url` — ссылка на страницу бронирования (для создателя — событие, для площадки — список бронирований).
