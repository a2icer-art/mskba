# task_init_description
нам нужно сперва спланировать интерфейсную часть по работе с добавленной сущностью расписание. во первых нужно будет завести новый раздел "расписание" для страницы зала - в sidebar в группу меню "общее" и саму страницу расписания

# task_short_description
Спланировать UI для расписания: пункт "Расписание" в sidebar (группа "Общее") и страница расписания зала.

# task_full_description
Нужно реализовать интерфейс управления расписанием площадки: добавить пункт "Расписание" в sidebar группы "Общее" на странице зала, создать отдельную страницу расписания, отобразить недельные интервалы и исключения, предусмотреть UX добавления/редактирования интервалов и перерывов. Интерфейс должен учитывать право `venue.schedule.manage` и отображать пустые состояния. Использовать существующие паттерны Presentation слоя, Inertia/Vue и модальные окна.

# scope (дополнительно)
- Навигация: добавить пункт в sidebar для площадки.
- Страница расписания: базовая структура, список интервалов и исключений.
- Формы: модальные окна для добавления/редактирования.
- Права доступа: скрывать действия без `venue.schedule.manage`.
- Документация: зафиксировать в `docs/project/current_state.md`.
