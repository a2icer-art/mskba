# task_full_description
Нужно расширить создание события: в попапе создания события добавить опциональный выбор площадки для игровых типов (игра, игровая тренировка, тренировка) с учетом того, что в будущем появятся неигровые типы. Если площадка выбрана, при отправке формы сначала валидируются даты события (как сейчас), затем проверяется расписание площадки и занятость для диапазона события. При успешной проверке создаются и событие, и бронирование, в одной транзакции. Если проверка расписания/занятости не проходит, выводится ошибка и не создается ни событие, ни бронирование. Пользователь должен иметь возможность очистить выбранную площадку и создать только событие.

# План выполнения задачи
1. UI: обновить попап создания события — убрать дефолтный тип, показать выбор площадки только для игровых типов, поддержать очистку выбора. (subtasks/01_ui.md)
2. Backend: принять venue_id, валидировать даты, при выбранной площадке создать событие + бронирование в транзакции с проверкой расписания/занятости и корректной ошибкой. (subtasks/02_backend.md)
3. Права/UX: учитываем право бронирования — если нет, скрываем выбор площадки и показываем понятную ошибку при попытке. (subtasks/03_permissions.md)
4. Документация: обновить описание флоу и статус задачи. (subtasks/04_docs.md)
