# task_full_description
Нужно расширить текущий адресный ввод (Yandex) так, чтобы при выборе адреса автоматически определялась станция метро и записывалась для площадки. Решение должно учитывать существующий интеграционный слой (address-suggest), структуру Address/Venue, и работать без ручного подбора, но оставлять возможность корректировки, если метро не найдено или определено неверно. Важно продумать источник данных (метро из Yandex или маппинг к нашей справочной таблице Metro) и правила сопоставления (по названию, линии, городу). Также нужно описать, как обрабатываются случаи без метро.

# План выполнения задачи
1. Проанализировать текущий поток ввода адреса и места хранения метро, определить точку расширения. (subtasks/01_analysis.md)
2. Спроектировать логику сопоставления метро из Yandex с локальным справочником и правила fallback. (subtasks/02_matching.md)
3. Реализовать backend-часть (интеграция, сохранение в Address/Venue) и обновить данные в выдаче. (subtasks/03_backend.md)
4. Обновить UI адресного ввода (автозаполнение метро, возможность коррекции) и локальные проверки. (subtasks/04_frontend.md)
5. Зафиксировать изменения в документации проекта. (subtasks/05_docs.md)
